SkoHub

Blog

SkoHub supports a novel approach for following publications about a specifc subject by upgrading Knowledge Organization Systems (KOS, sometimes also called “controlled vocabularies”) encoded in SKOS to content hubs. This is done by extending SKOS-enchoded KOS current web standards for sending and subscribing to notifications (Linked Data Notifications & ActivityPub).

The project has four deliverables. While working on the backend infrastructure for receiving and pushing notifications (SkoHub PubSub), SkoHub also provides means to publish a controlled vocabulary (SkoHub Vocabs) as well as an editor to describe web resources and send notifications (SkoHub Editor), even from within the browser (skohub-extension).

Repositories

skohub-pubsub

Inboxes and subscriptions

SkoHub Pubsub provides the SkoHub core infrastructure, setting up basic inboxes for subjects plus the ability of subscribing to push subscriptions for each new notification.

skohub-vocabs

SkoHub Vocabs is atatic site generator for Simple Knowledge Organization Systems

This part of the project covers the need to easily publish a controlled vocabulary as a SKOS file, with a basic lookup API and a nice HTML view including links to an inbox for each subject. For a walkthrough, see the blog post.

Code

skohub-editor

Describing & linking learning resources, sending notifications

The editor will run in the browser and enable structured description of educational resources published anywhere on the web. It includes validation of the entered content for each field and lookup of controlled values via the API provided by skohub-ssg.
